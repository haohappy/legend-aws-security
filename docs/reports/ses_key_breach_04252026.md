# 安全事件报告：lending_ses_prod Access Key 泄露

**日期**: 2026-04-25  
**严重等级**: 高  
**状态**: 已处置  

## 1. 事件概述

IAM 用户 `lending_ses_prod` 的 Access Key 被外部攻击者利用，在 `us-west-2` 区域执行了 EC2 相关操作。该用户原用途为 SES 邮件发送服务。

## 2. 受影响资产

| 项目 | 详情 |
|------|------|
| IAM 用户 | `lending_ses_prod` |
| Access Key ID | `AKIAZNVMXQFBROVA2FP7` |
| AWS 账户 | `647828570435` |
| Key 创建时间 | 2023-01-19 |
| Key 存续时间 | 1191 天（未轮换） |

## 3. 攻击者活动

通过 CloudTrail 确认，攻击者活动仅限 `us-west-2` 区域：

| 时间 (UTC+5) | API 调用 | 源 IP | 说明 |
|--------------|----------|-------|------|
| 2026-04-25 02:47:12 | `ImportKeyPair` | 216.126.225.20 | 导入 SSH 密钥对，为启动 EC2 做准备 |
| 2026-04-25 02:47:13 | `DescribeInstances` | 216.126.225.20 | 探测现有 EC2 实例 |

攻击者 IP `216.126.225.20` 非本组织 IP。攻击模式符合典型的被盗凭据启动挖矿实例流程。

**其他 14 个 region 扫描结果**：全部无异常活动。

## 4. 影响评估

- 攻击者未成功启动 EC2 实例
- 未发现残留资源（无实例、无 key pair）
- 未发现数据泄露迹象
- **实际损失：无**

## 5. 处置措施

| 时间 | 操作 |
|------|------|
| 2026-04-25 05:06 | 通过 `aws-user-audit.sh` 发现异常（最后使用服务为 ec2，区域 us-west-2） |
| 2026-04-25 05:07 | CloudTrail 确认攻击者 IP 和操作 |
| 2026-04-25 05:08 | 禁用 Access Key (`Inactive`) |
| 2026-04-25 05:09 | 全 region 扫描确认攻击范围 |

## 6. 根因分析

- Access Key 创建于 2023-01-19，**从未轮换**
- 用户**未启用 MFA**
- 用户**未配置 IP 限制策略**
- Key 泄露渠道尚未确认（可能为代码仓库、日志、配置文件等）

## 7. 后续建议

1. **排查泄露渠道**：搜索代码仓库、CI/CD 日志、配置文件中是否残留该 Key
2. **创建新 Key**：如业务仍需使用，创建新 Key 并附加 `DenyWithoutIPorMFA` 策略
3. **权限收敛**：审查 `lending_SES` 策略，确保仅含 SES 相关权限，移除 EC2 等无关权限
4. **全面审计**：对账户下所有 IAM 用户执行同样的审计流程
5. **建立轮换机制**：Access Key 最长 90 天轮换
